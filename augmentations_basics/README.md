# Примечания к решению

## Датасет

Оригинальный датасет "герои" не был предоставлен/выложен для этого потока
(ссылка на архив была недоступна на момент выполнения ДЗ), поэтому в качестве
временной замены использован **STL10** — датасет с похожей структурой
(объекты/классы), позволяющий продемонстрировать всю логику аугментаций и
пайплайна без изменения кода `CustomImageDataset`.

Использовано 5 классов, по 80 изображений train / 15 val / 5 test на класс
(см. ячейку загрузки датасета в ноутбуке).

Из-за этого в **Задании 3** распределение размеров изображений вырождается
в одну точку на графике — все изображения STL10 имеют фиксированный размер
96×96. На реальном датасете героев (с изображениями разного размера) график
показал бы содержательный разброс по ширине/высоте.

## Задание 6 (дообучение модели)

Валидационная выборка небольшая (75 изображений, по 15 на класс), поэтому
кривые train/val loss и accuracy могут выглядеть шумными — это ожидаемо при
таком размере val-сета, а не ошибка обучения.

## Запуск

Ноутбук рассчитан на запуск в Google Colab, монтирует Google Drive для
сохранения промежуточных данных. Все результаты (графики по Заданиям 1–6)
также сохраняются и закоммичены в папку `results/` в корне репозитория.
